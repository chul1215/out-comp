### 1. 프로젝트 목적

1. **2025년 외래환자 데이터를 기반으로 2026년 마케팅 타겟 진료과 및 홍보 아이템을 선정**하는 분석 도구 구축
2. **월별/분기별/계절별** 외래환자 추이를 파악하여 시기별 마케팅 전략 수립 지원
3. **복합 점수(성장률 + 규모 + 신환비율)** 기반 다면적 평가로 마케팅 우선순위 도출
4. **YoY(전년 대비) 비교**: 25년(내장 데이터) vs 26년(업로드 데이터) 자동 비교
5. 대전선병원/유성선병원 **병원별 독립 분석** 및 브리핑 자료 자동 생성

> **v2.0 변경 사항 (2026-02-22)**
> - 25년도 데이터(진료실적 + 진단별 통계) 프로그램 내 자체 식립 → 업로드 불필요
> - 26년도 데이터만 업로드하여 분석
> - YoY 비교: 25년(내장) vs 26년(업로드)으로 자동 구성
> - 서버 불필요: HTML 파일 더블클릭으로 바로 실행 (CDN 라이브러리 인라인 내장)

### 2. 대상 기간 정의

- **월별:** 1월 ~ 12월 (12개월 개별 분석)
- **분기별:** Q1(1~3월), Q2(4~6월), Q3(7~9월), Q4(10~12월)
- **계절별:** 봄(3~5월), 여름(6~8월), 가을(9~11월), 겨울(12~2월)
  - 겨울 시즌은 동일 연도 기준(12월+1월+2월)으로 산출
  - 2026년 데이터 추가 시 크로스연도(전년 12월 + 당해 1~2월) 옵션 제공 예정
- (확장 옵션) 사용자가 임의 기간을 선택하여 분석 가능

### 2-1. 제외 진료과 정의

> 마케팅 분석 대상에서 제외하는 진료과 코드 규칙. 데이터 로드 시 자동 필터링 적용.

| # | 제외 규칙 | 설명 | 예시 |
|---|----------|------|------|
| 1 | `DN`으로 시작하는 코드 | 치과 관련 진료과 | DN1, DN2, DNS 등 |
| 2 | `HPC` | 건강검진센터 | HPC |
| 3 | `FM` | 가정의학과 | FM |
| 4 | 코드 끝에 `2`가 붙는 코드 | 미수(수납 미완료) 코드 | GS2, IM2 등 |

- 개발 과정에서 **진료과 코드 전체 목록을 선행 학습**하여 제외 대상을 사전 확인받는 절차 필요
- 제외된 진료과는 모든 집계/점수/차트/랭킹에서 일괄 제외
- 제외 현황은 데이터 업로드 후 요약 화면에 표시 (예: "5개 진료과 제외됨")

### 4. 지표 정의 (산출물)

#### 4.1 기본 지표

| 지표 | 산출식 | 단위 |
|------|--------|------|
| 총 환자수 | 계(합계) | 명 |
| 환자유형별 | 신환 / 초진 / 재진 | 명 |
| 보험유형별 | 일반 / 보험 / 보호 / 자보 / 산재 / 기타 | 명 |
| 성별 분포 | 남자 / 여자 | 명 |
| 진료형태별 | 선택진료 / 일반진료 | 명 |
| 신환비율 | 신환 / (신환 + 초진 + 재진) x 100 | % |
| 진단별 건수 | 진단코드별 건수 합계 | 건 |

#### 4.2 복합 점수 (마케팅 타겟 선정용)

- 성장률, 규모, 신환비율을 min-max 정규화 후 가중 합산
- 가중치 기본값: 성장률 40% + 규모 30% + 신환비율 30%
- 사용자가 슬라이더로 가중치 조절 가능

---

### 5. 복합 점수 산출 공식

```
1. Growth_Score (성장률 점수)
   성장률 = (당기 환자수 - 전기 환자수) / 전기 환자수 x 100
   Growth_Score = minmax_normalize(성장률) x 100

2. Volume_Score (규모 점수)
   Volume_Score = minmax_normalize(총 환자수) x 100

3. NewPatient_Score (신환비율 점수)
   신환비율 = 신환 / (신환 + 초진 + 재진) x 100
   NewPatient_Score = minmax_normalize(신환비율) x 100

4. Composite Score (복합 점수)
   Composite = 0.40 x Growth_Score + 0.30 x Volume_Score + 0.30 x NewPatient_Score

※ minmax_normalize(x) = (x - min) / (max - min)
※ 전기 데이터 없는 경우(신규 진료과 등) 성장률 = 0 처리
```

### 6. 기능 요구사항

#### 6.1 데이터 로드 및 처리

- **25년도 데이터**: 프로그램 내 자체 식립 → 업로드 불필요, 프로그램 시작 시 즉시 분석 가능
- **26년도 데이터**: 엑셀(.xlsx) 파일 업로드 (SheetJS 라이브러리 활용, 로컬 처리)
- 병원별(대전선/유성선) 데이터 독립 로드
- 월별 파일 자동 병합 및 기간 구분 자동 분류 (월별/분기별/계절별)
- 필수 컬럼 검증 및 오류 메시지 표시
- **실행 방식**: 단일 HTML 파일 더블클릭으로 바로 실행 (별도 서버 불필요, CDN 라이브러리 인라인 내장)

#### 6.2 병원 전환

- 탭 또는 토글 방식으로 대전선병원/유성선병원 전환
- 한 화면에 한 병원 집중 분석 (UI 단순화)

#### 6.3 기간 선택

- 월별(1~12월) / 분기별(Q1~Q4) / 계절별(봄/여름/가을/겨울) 전환
- 기간 변경 시 모든 지표 실시간 갱신

#### 6.4 마케팅 타겟 추천

- 복합 점수 기반 진료과/진단 TOP 10 자동 추천
- 가중치 슬라이더로 성장률/규모/신환비율 비중 조절
- 자동 인사이트 문구 생성 (템플릿 기반)

#### 6.5 YoY 비교

- **25년(내장) vs 26년(업로드)** 자동 비교
- 26년 데이터 미업로드 시: YoY 탭 비활성화 또는 안내 메시지 표시
- 비교 지표: 총 환자수 + 환자유형별(신환/초진/재진) + 진단별 건수
- 증감률 및 증감 건수 산출

#### 6.6 브리핑 자료 생성

- 마크다운(.md) 형식의 브리핑 요약 자동 생성
- 핵심 인사이트, TOP 진료과/진단, 추천 마케팅 아이템 포함

#### 6.7 데이터 다운로드

- 분석 결과 테이블 엑셀 다운로드 지원

    - 분기별 Grouped Bar Chart

    - 복합 점수 가중치 슬라이더

5. **YoY 비교**
    - 증감 KPI 카드 (총 환자수/신환/초진/재진)

    - 진단별 증감 테이블
6. **진단 상세**
    - 진료과별 진단 TOP 20
    - 의사별 진단 분포

### 8. 검증 체크리스트

- 제작 전 체크리스트
    - 데이터 컬럼명이 실제 엑셀 파일과 일치하는가?
        - 진단별 통계: 과코드, 입력자ID, 입력자명, 진단코드, 진단명, 건수 ✅
        - 진료실적: 진료과별, 일반, 보험, 보호, 자보, 산재, 기타, 신환, 초진, 재진, 남자, 여자, 선택진료, 일반진료, 계 ✅
    - 제외 진료과 규칙(DN*, HPC, FM, *2)이 정확히 적용되는가? → ✅
    - 병원 비교 방식은 탭/토글 전환인가? → ✅
    - 복합 점수 가중치 기본값은 성장률 40% + 규모 30% + 신환비율 30%인가? → ✅
    - 겨울 시즌 기준은 동일 연도(12+1+2월)인가? → ✅
    - YoY 비교에 총 환자수 + 환자유형별 + 진단별 건수 3개 모두 포함되는가? → ✅
- 1차 브리핑 후 체크리스트
    - (정합성) 진료과별 합계 = 전체 합계 일치 여부
    - (분모) 환자수 0인 경우 처리 규칙 적용 확인
    - (저표본) 극소 건수 진료과/진단의 복합 점수 왜곡 방지 확인
    - (해석) 마케팅 추천 결과의 실무 활용 가능성 검증
