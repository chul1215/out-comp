> **문서 버전:** v2.0
> **작성일:** 2026-02-20 (최종 수정: 2026-02-22)
> **상태:** Draft
> **관련 부서:** 마케팅팀 / 기획팀

> **v2.0 변경 사항 (2026-02-22)**
> - 25년도 데이터 프로그램 내 자체 식립 → 업로드 불필요

## 1. 개요 (Overview)

### 1.1 배경

대전선병원과 유성선병원은 외래환자 데이터를 체계적으로 분석하여 마케팅 전략을 수립할 필요가 있다. 현재 월별 외래 진료 실적과 진단별 통계가 엑셀 파일로 축적되고 있으나, 이를 종합적으로 분석하여 마케팅 타겟을 도출하는 도구가 부재한 상황이다. 2025년 데이터를 기반으로 2026년 마케팅 방향을 설정하기 위한 분석 프로그램이 필요하다.

### 1.2 목적

1. 2025년 외래환자 데이터를 기반으로 **2026년 마케팅 타겟 진료과 및 홍보 아이템 선정**
2. **월별/분기별/계절별** 외래환자 추이 분석으로 시기별 마케팅 전략 지원
3. **복합 점수(성장률 + 규모 + 신환비율)** 기반 다면적 평가로 마케팅 우선순위 도출
4. **YoY 비교**: 25년(내장) vs 26년(업로드) 자동 비교로 성장/감소 진료과 및 진단 식별

### 1.3 대상 사용자

| 사용자 | 역할 | 주요 니즈 |
|--------|------|-----------|
| 마케팅팀 | 데이터 업로드, 타겟 분석, 브리핑 자료 생성 | 마케팅 대상 진료과/진단 선정 및 근거 자료 |
| 병원 경영진 | 분석 결과 열람, 의사결정 | 요약 대시보드 및 핵심 인사이트 |
| 기획팀 | 마케팅 기획안 수립 | 시기별 트렌드 및 타겟 추천 데이터 |

### 1.4 용어 정의

| 용어 | 정의 |
|------|------|
| 신환 | 해당 병원에 처음 내원한 환자 |
| 초진 | 해당 진료과에 처음 방문한 환자 |
| 재진 | 해당 진료과에 재방문한 환자 |
| 신환비율 | 신환 / (신환 + 초진 + 재진) x 100 |
| 복합 점수 | 성장률(40%) + 규모(30%) + 신환비율(30%) 가중 합산 점수 |
| YoY | Year-over-Year, 전년 동기 대비 비교 |
| 진단별 통계 | 진료과-의사-진단코드 단위의 외래 진단 건수 데이터 |
| 진료실적 | 진료과 단위의 환자유형별/보험유형별 외래 실적 데이터 |

### FR-01. 데이터 로드

| 항목 | 상세 |
|------|------|
| **ID** | FR-01 |
| **우선순위** | P0 (필수) |
| **설명** | 25년도 데이터는 프로그램 내 자체 식립, 26년도 데이터만 사용자가 업로드한다 (SheetJS 활용, 서버 불필요) |

**인수 조건 (Acceptance Criteria):**
- [ ] **25년도 데이터 자체 식립**: 프로그램 실행 시 즉시 로드 (업로드 불필요)

  - 진단별 통계 데이터 내장
- [ ] **26년도 데이터 업로드**: 엑셀(.xlsx) 파일 형식 지원
- [ ] 26년도 2종 데이터 각각 업로드: 외래 진단별 통계 + 외래환자 진료실적
- [ ] 진단별 통계 필수 컬럼 검증: 과코드, 입력자ID, 입력자명, 진단코드, 진단명, 건수
- [ ] 진료실적 필수 컬럼 검증: 진료과별, 일반, 보험, 보호, 자보, 산재, 기타, 신환, 초진, 재진, 남자, 여자, 선택진료, 일반진료, 계
- [ ] 필수 컬럼 누락 시 구체적 오류 메시지 표시
- [ ] 26년도 파일 업로드 성공 시 데이터 건수 미리보기 표시
- [ ] 병원별(대전선/유성선) 독립 데이터 로드
- [ ] 26년도 데이터 미업로드 상태에서도 25년 분석 화면 정상 작동

### FR-06. 복합 점수 산출

| 항목 | 상세 |
|------|------|
| **ID** | FR-06 |
| **우선순위** | P0 (필수) |
| **설명** | 마케팅 타겟 선정을 위한 복합 점수를 산출한다 |

**산출 로직:**

```
1. Growth_Score = minmax_normalize(성장률) x 100
   성장률 = (당기 환자수 - 전기 환자수) / 전기 환자수 x 100

2. Volume_Score = minmax_normalize(총 환자수) x 100

3. NewPatient_Score = minmax_normalize(신환비율) x 100
   신환비율 = 신환 / (신환 + 초진 + 재진) x 100

4. Composite = 0.40 x Growth + 0.30 x Volume + 0.30 x NewPatient
```

**인수 조건:**
- [ ] 진료과 단위 및 진단 단위 각각 복합 점수 산출
- [ ] 가중치 슬라이더로 성장률/규모/신환비율 비중 실시간 조절 가능
- [ ] 슬라이더 합계는 항상 100% 유지
- [ ] 가중치 변경 시 순위 실시간 갱신
- [ ] 전기 데이터 없는 경우(신규 진료과) 성장률 = 0 처리

## 5. 복합 점수 산출 규칙 상세

### 5.1 정규화 규칙

```
minmax_normalize(x) = (x - min) / (max - min)

- min, max는 해당 기간 내 전체 진료과(또는 진단)의 값 중 최솟값/최댓값
- max = min인 경우 정규화 값 = 0 처리
```

### 5.2 예외 처리

```
IF 전기 데이터 없음 (신규 진료과) THEN 성장률 = 0
IF 전기 환자수 = 0 THEN 성장률 = 0
IF (신환 + 초진 + 재진) = 0 THEN 신환비율 = 0
```

### 5.3 가중치 조절 규칙

- 기본 가중치: 성장률 40%, 규모 30%, 신환비율 30%
- 슬라이더로 0~100% 범위에서 조절 가능
- 3개 슬라이더 합계는 항상 100% 유지
- 하나의 슬라이더 조절 시 나머지 2개가 비례 조정

### 5.4 비교 기간 매칭 규칙

- 월별 비교: 동일 월 전기 vs 당기 (예: 2024년 3월 vs 2025년 3월)
- 분기별 비교: 동일 분기 전기 vs 당기
- 계절별 비교: 동일 계절 전기 vs 당기
